# Pair Aura Web V5 复盘报告：分享链路与二维码完善

## 1. 本轮目标

本轮升级集中在三个方面：

1. 完善二维码海报，使它明确指向当前报告链接，并在二维码生成失败时给出可复制链接兜底。
2. 完善分享链接体系，解决本地预览链接不能直接发给别人的问题。
3. 从项目源设想出发，重新检查 V4 到 V5 的全流程是否仍符合“轻量、可分享、可视化、年轻化、可落地”的方向。

## 2. 与项目源设想的符合度

### 符合点

- 产品仍然是“双人关系分析工具”，不是重新做测评平台。
- 输入仍保持轻量：关系组、语气、A/B 人格类型。
- 报告仍以合拍指数、关系类型、相似/互补/摩擦、Top 差异和短文案为主。
- 分享仍以二维码海报、复制链接和系统分享为核心。
- 不引入登录、数据库、后端或付费机制，适合冷启动验证。

### 边界控制

- 报告仍保留娱乐提示，不作心理诊断、职业筛选或关系判决。
- 分享链接只编码用户选择的类型、关系组和语气，不上传个人数据。
- 本地预览链接不会被包装成可用线上链接，而是明确提示需要部署后填写公开域名。

## 3. 本轮实际改动

### 3.1 分享链接设置区

在分享海报页新增“分享链接设置”：

- 公开首页地址输入框，例如 `https://your-domain.com/`。
- 应用域名按钮。
- 当前报告分享链接只读框。
- 推荐分享文案只读框。
- 分享状态提示。

如果用户在本地预览，页面会提示：当前是本地链接，发给别人打不开；部署后填写线上域名再分享。

### 3.2 链接生成逻辑

新增两条链接逻辑：

- `buildLocalReportUrl`：用于当前浏览器地址栏，保证本机测试能恢复报告。
- `buildReportUrl`：用于公开分享，优先使用用户填写的线上域名。

这样本地测试和线上分享不会互相冲突。

### 3.3 二维码生成增强

二维码生成现在会：

- 优先使用已加载的 QRCode 库。
- 如果未加载，会尝试从备用 CDN 动态加载。
- 如果仍失败，会在海报中写入兜底提示，并引导用户复制链接。
- 二维码状态会显示“已生成”“本地预览链接”或“加载失败”。

### 3.4 分享按钮增强

新增：

- 复制链接。
- 复制完整分享文案。
- 系统分享。
- 打开报告链接。
- 下载海报失败时提示可长按保存或截图分享。

## 4. 全流程检查

### 4.1 输入流程

已检查：

- 首页进入输入页正常。
- 关系组为 2 个。
- 报告语气为 2 个。
- A/B 可选择 MBTI 或 SBTI。
- MBTI 保留 16 型。
- SBTI 保留 27 型。
- MBTI 细分指标保留 9 种。

### 4.2 报告生成流程

已用 Node VM 跑通以下组合：

1. MBTI × MBTI：ENFP 表达外放版 × ISTJ 计划强化版。
2. SBTI × SBTI：LOVE-R × CTRL。
3. MBTI × SBTI：INTJ 稳定版 × HHHH。

检查结果：

- 每个组合都能生成关系类型。
- 合拍指数、相似度、互补度、摩擦值均有输出。
- 每份报告有 3 张核心文案卡。
- 每份报告有 2 张场景展开卡。
- 每份报告有 3 个算法差异重点。
- 分享文案可生成。

### 4.3 分享流程

已检查：

- 报告链接包含 `?r=` 参数，可还原报告输入。
- 本地链接和公开链接分离。
- 填写公开域名后，分享链接会使用公开域名。
- 分享文案会包含报告标题、分数和链接。
- 二维码海报与复制链接使用同一个报告 URL。

### 4.4 技术质量检查

- `index.html` 仍为单文件结构，可直接打开。
- CSS 和 JS 均内嵌。
- JavaScript 语法检查通过。
- 无后端依赖。
- 无登录依赖。
- 无数据库依赖。
- URL 编码和解码逻辑通过测试。
- 旧报告链接参数仍由现有 decode 逻辑处理。

## 5. 当前仍存在的限制

1. 二维码依赖浏览器联网加载 QRCode 库；离线情况下会显示兜底提示，而不是生成真正可扫的二维码。
2. 纯静态网页无法自动知道最终线上域名；本地测试时需要手动填写部署后的公开地址。
3. 微信内分享卡片标题、缩略图和描述不能完全自定义；要做到微信卡片级分享，需要后续接微信公众号 JS-SDK 和后端签名。
4. 当前分享链接把输入信息编码在 URL 中，方便无后端运行，但如果未来加入更多字段，URL 会变长，需要后端短链或报告 ID。

## 6. 下一步优化方向

### 优先级 A：正式传播能力

- 部署到真实 HTTPS 域名。
- 配置默认公开域名，用户无需手动填写。
- 做一个短链服务或报告 ID，缩短 URL。
- 为微信、小红书、浏览器预览补充 Open Graph 元信息。

### 优先级 B：报告质量

- 加入 6 个轻量关系行为问题：回复频率、吵架方式、安慰偏好、边界感、计划方式、表达风格。
- 基于用户反馈“准/不准”继续调算法权重。
- 为不同关系类型增加更多专属标题和文案。

### 优先级 C：输入增强

- 支持九型人格作为第三个入口。
- 支持 MBTI 维度百分比输入。
- 支持 SBTI 15 维度截图识别，但必须先做识别确认页。

### 优先级 D：长期产品化

- 用户反馈按钮：准 / 不准 / 想发给 TA。
- 数据埋点：完成率、分享率、复制链接率、海报下载率。
- 主题报告包：情侣、闺蜜、室友、项目组、旅行搭子。
- 多人关系图谱：宿舍、社团、小组项目。

## 7. 结论

V5 已经更接近可冷启动测试的版本：

- 输入轻。
- 报告短。
- 算法重点清楚。
- 文案有传播感。
- 分享链路有二维码、链接、文案三种出口。
- 本地测试和线上分享的边界更清晰。

下一步最值得做的是：部署到真实域名，补充用户反馈和数据埋点，然后再考虑截图识别与 AI 个性化。
