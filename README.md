# Pair Aura V14 单文件版

V14 基于 V13 继续优化两件事：

1. **首页定位重做**：首页不再偏情侣/暧昧，而是中性表达“朋友、情侣、搭子、项目”都能用；突出产品核心是“对比”和“关系翻译”。
2. **透明匿名数据回传设计**：拒绝隐藏后门，改成用户可见、可取消的匿名数据授权。只有配置 `ANALYTICS_ENDPOINT` 后才会向你的接口回传匿名报告数据和关键操作事件。

## 使用方式

GitHub Pages 直接替换仓库里的 `index.html` 即可。

## 保留特性

- 单文件部署，不依赖 `styles.css` / `app.js`。
- 支持 MBTI、A/T、SBTI。
- 关系组 2 个：友谊/情侣、搭子/学习/项目。
- 语气 2 个：温情吐槽、现实建议。
- 每份报告固定 6 张特质细节评价卡。
- 关系签文库：古代诗词 22 条 + 现代歌单签 24 条。
- 分享链接可复制到微信，海报右下角生成二维码。

## 匿名数据回传设置

默认情况下不会发送数据，因为 `ANALYTICS_ENDPOINT` 为空。

如需收集真实使用报告：

1. 自己准备一个接收 POST JSON 的接口，例如 Google Apps Script、Supabase Edge Function、Cloudflare Worker。
2. 打开 `index.html`，找到：

```js
const ANALYTICS_ENDPOINT = '';
```

3. 改成你的接口地址：

```js
const ANALYTICS_ENDPOINT = 'https://your-endpoint.example.com/collect';
```

4. 保留页面上的用户授权提示，不要隐藏。

回传内容只包括：类型、关系组、语气、分数、关键词、Top 差异、诗词/歌单签、点击事件、屏幕尺寸。不记录昵称、联系方式、聊天内容或原始截图。

## 分享说明

- 点击“复制报告链接”可把当前报告链接复制到微信。
- 点击“复制分享文案”可复制一段完整分享文字。
- 点击“发给 TA”会优先调用系统分享，失败时自动复制分享文案。
- 点击“打开检查”会用新窗口打开当前报告链接，方便确认报告能还原。
- 下载海报会等待二维码绘制完成后再下载，尽量保证二维码可扫码。
