# Pair Aura Web V4 修改复盘

## 修改目标

V4 的目标是降低用户选择成本，同时提升报告差异度和语言吸引力：

- 把关系场景压缩为两个大入口：友谊/情侣、搭子/学习/项目。
- 把报告语气压缩为两个高差异入口：温情吐槽、现实版。
- 在报告中重新展开细分场景，用朋友/情侣或学习/项目两张场景卡补回语境差异。
- 缩短报告文字，突出合拍指数、相似度、互补度、摩擦值和最大差异 Top 3。
- 新增 copy engine，根据最大差异维度生成更有活人感的年轻化文案。

## 与项目设想的一致性

本次修改仍然遵守产品源中的核心定位：Pair Aura 不是测评平台，而是双人关系解释器。V4 没有增加测试题和复杂注册流程，只优化输入选择、报告结构和传播文案。

## 功能改动

### 1. 关系场景

旧版：

- 情侣/暧昧
- 好朋友/闺蜜
- 室友/生活搭子
- 学习/项目搭档

新版：

- 友谊 / 情侣
- 搭子 / 学习 / 项目

旧链接兼容：romantic、friend、roommate 会自动映射到 close；teamwork 会自动映射到 collab。

### 2. 报告语气

旧版：

- 温柔分析版
- 有梗吐槽版
- 犀利现实版

新版：

- 温情吐槽
- 现实版

旧链接兼容：warm、meme 会自动映射到 warm_roast；sharp 会自动映射到 reality。

### 3. 报告结构

新版报告顺序：

1. 关系类型 + 合拍指数
2. 一句传播型 punchline
3. 相似度 / 互补度 / 摩擦值
4. A/B 类型概览
5. 算法重点：最大差异 Top 3 + 关键维度条
6. 场景展开卡：朋友/情侣或学习/项目
7. 三张出圈文案卡：最像的地方、最容易卡住、一句建议
8. 分享海报入口

- CSS、JS 均内嵌，无需额外打包。
- JavaScript 语法检查通过。

## 风险边界

- 文案更尖锐，但不做关系审判。
- 保留“仅供娱乐、社交破冰和自我理解”的免责声明。
- 不增加用户数据存储，不涉及登录、手机号或隐私截图。
