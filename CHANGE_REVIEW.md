# Web 版本开发复盘

## 1. 入口复盘

目标：从微信小程序入口改为网页入口。

完成情况：已完成静态 H5 版本，浏览器、微信、QQ、小红书、二维码都可进入。

判断：更适合 MVP 验证，因为不受微信小程序审核和入口限制。

## 2. 功能复盘

已完成：

- 首页；
- 输入页；
- 关系场景选择；
- 报告语气选择；
- A/B 类型输入；
- 报告生成；
- 报告链接分享；
- 二维码海报生成；
- 下载海报；
- 系统分享。

符合“轻、快、可分享”的第一版要求。

## 3. 算法复盘

仍然采用规则算法：

- 人格类型映射到统一向量；
- 计算相似度、互补度、冲突风险；
- 根据关系场景加权；
- 输出关系类型和解释卡片。

这比把分数直接交给 AI 更稳定、更可解释。

## 4. 传播复盘

Web 版的传播链路是：

```text
生成报告
↓
复制链接 / 生成二维码海报
↓
分享到微信、小红书、朋友圈、QQ、社群
↓
对方点开同一份报告
↓
重新输入或反测
```

这比小程序更开放，但微信内高级分享能力需要 JS-SDK。

## 5. 技术复盘

优点：

- 无后端；
- 无数据库；
- 无登录；
- 可以直接部署；
- 成本接近 0；
- 代码结构简单，新手可维护。

风险：

- URL 参数较长；
- 无数据统计；
- 无用户历史；
- 二维码库依赖 CDN；
- 微信自定义分享卡片需要额外后端。

## 6. 下一步

优先顺序：

1. 替换正式品牌名和域名；
2. 增加真实小红书风格海报模板；
3. 加访问统计；
4. 加短链接；
5. 加用户反馈按钮：“准 / 不准 / 想发给 TA”；
6. 再考虑 OCR 和 AI 个性化报告。
