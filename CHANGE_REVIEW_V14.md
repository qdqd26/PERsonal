# Pair Aura V14 修改复盘

## 1. 修改目标

本轮基于 V13 做两类升级：

1. 首页重新设计：避免过度偏情侣/暧昧，兼顾朋友、搭子、学习和项目合作场景，同时突出产品的对比感、人情味和关系翻译特色。
2. 数据回传能力设计：用户希望获得真实使用检测报告，但不能做隐藏后门，因此改成透明、可取消、匿名化的数据回传机制。

## 2. 与项目源设想的一致性

项目源强调 Pair Aura 不是新的测评平台，而是双人关系分析工具：用户输入两个人已有的人格结果，系统生成匹配度、差异点、互补点、潜在冲突和相处建议。V14 的首页文案从“你和 TA 会发生什么”升级为“相似、差异、互补和容易误会的地方”，更突出对比与关系解释。

项目源也明确要求不要做性格审判或心理诊断。V14 的数据回传不是隐藏后门，而是可见授权、匿名化、可关闭的产品分析能力。

## 3. 首页改动

旧首页偏情侣表达：

- 你和 TA，是哪种关系气场？
- 不是给你贴标签，是看你和 TA 会发生什么。

新版首页改为：

- 两个人之间，会发生什么？
- 不只看你是谁，也看你们如何相处。
- 输入两个人的 MBTI / SBTI，把相似、差异、互补和容易误会的地方翻译成一份有人情味的关系说明书。

新增三句悬浮锐评：

- 不是不合，是一个在等回应，一个在等空间。
- 合作最怕的不是能力差，是默认对方会自动懂。
- 有些关系的关键，不是更像，而是愿意互相翻译。

这些文案覆盖朋友、情侣、搭子、项目关系，不再只偏暧昧。

## 4. 文案调整

继续去除 AI 味表达，避免“说人话就是”“谁更正常”等生硬或伤人的句式。改成更柔和、更有人情味的表达：

- 你们不用争谁更对，把彼此的需要说清楚，关系就会轻一点。
- 话说出来之前，先让对方知道你不是来攻击的。
- 这段关系最需要的不是证明谁在乎，而是少一点猜测，多一点坦白、也多一点温柔。

## 5. 分享链路增强

新增：

- 当前报告链接只读框。
- “打开检查”按钮，用于确认报告链接能在新窗口还原。
- 下载海报前等待二维码绘制完成。
- 分享链接、分享文案、系统分享、海报下载均记录匿名操作事件。

## 6. 匿名数据回传设计

新增：

```js
const ANALYTICS_ENDPOINT = '';
```

默认不发送任何数据。配置接口后，且用户未取消勾选时，才会发送匿名数据。

记录内容：

- 事件名称：report_generated、report_viewed、copy_report_link、copy_share_text、native_share_success、download_poster、qr_generated_success、qr_generated_fail、open_report_link。
- 匿名 sessionId。
- 输入类型：MBTI、A/T、SBTI、关系组、语气。
- 结果数据：关系类型、合拍指数、相似度、互补度、风险值、关键词、Top 差异、签文类型。
- 设备信息：视口宽高。

不记录：

- 昵称原文。
- 手机号、邮箱、微信号。
- 聊天记录。
- 原始截图。
- 任何可直接识别真实身份的信息。

## 7. 质量检查

完成以下检查：

- index.html 仍为单文件。
- CSS 完整内嵌。
- JS 完整内嵌。
- JavaScript 语法检查通过。
- MBTI × MBTI 报告生成通过。
- MBTI × SBTI 报告生成通过。
- SBTI × SBTI 报告生成通过。
- 每份报告固定 6 张故事卡。
- 报告链接编码 / 解码通过。
- 匿名 analytics payload 不包含昵称原文。
- 分享链接预览可生成。
- 海报下载前会等待二维码绘制。
- 文案中无“说人话就是”。

## 8. 后续建议

如果要真正获得用户真实使用数据，下一步建议用 Google Apps Script 或 Supabase 做一个简单的 POST 接口，把匿名事件写入表格或数据库。不要用隐藏后门收集数据，必须保留页面提示和退出选项。
